# Clean-Room PR Checklist

在合并任意 `fx-superpowers` 相关变更前，逐项确认：

- [ ] 本次实现依据的是公开行为规格，而非上游实现代码。
- [ ] 未复制上游源码、脚本、模板文本到本仓库。
- [ ] 变更关联了 `docs/cleanroom/spec.md` 中明确条目。
- [ ] 新增功能有黑盒测试（至少 1 个成功路径）。
- [ ] 新增功能有失败路径测试（至少 1 个失败路径）。
- [ ] 变更更新了 `docs/cleanroom/feature-matrix.md`。
- [ ] 若存在行为差异，已在矩阵中标注 `Partial` 或 `Not Planned` 并说明原因。
- [ ] 命令支持 `--dry-run` 时，已验证无副作用。
- [ ] 文件写入路径有冲突策略（skip/overwrite/backup）之一。
- [ ] 文档中未出现对上游内部实现的引用或依赖。
